Project Overview & Objectives: The project aims to conduct a thorough analysis of Blinkit's sales performance, customer satisfaction, and inventory distribution. The final product is a dynamic, interactive dashboard that allows users to filter data by item type, outlet size, location, and fat content to uncover operational insights.

Key Metrics & KPIs
   1)The dashboard tracks four primary Key Performance Indicators (KPIs).
   2)Total Sales: The overall revenue generated from all sold items. 
   3)Average Sales: The average revenue per individual sale.
   4)Number of Items: The total count of unique items sold.
   5)Average Rating: The mean customer satisfaction score.

Core Visualizations & Analysis: The project breaks down analysis into several key charts 
  1)Product Fat Content: A donut chart analyzing how fat content (Low Fat vs. Regular) impacts total sales.
  2)Item Type Analysis: A bar chart showing sales performance across categories like fruits, vegetables, snacks, and household items.
  3)Outlet Establishment: A line chart identifying which year's outlet openings (e.g., 2018) contribute most to current revenue.
  4)Outlet Segmentation: Analysis of sales based on Size (Small, Medium, High) and Location (Tier 1, Tier 2, Tier 3 cities).
  5)Outlet Type Matrix: A comprehensive table comparing Supermarkets vs. Grocery Stores across all major metrics.

Technical Skills Covered
  1)Data Cleaning: Using Power Query to fix discrepancies (e.g., standardizing "LF" to "Low Fat")].
  2)DAX Calculations: Writing measures for Total Sales (SUM), Average Sales (AVERAGE), and Item Counts (COUNTROWS).
  3)Dynamic Parameters: Creating Field Parameters that allow the user to toggle an entire chart between Sales, Ratings, and Item Counts with one click.
  4)Interactive Design: Setting up "Edit Interactions" to ensure charts filter each other rather than just highlighting data.
